Architecture

# Pipeline — runs daily at 4:30 AM Dropbox PDFs │ ▼ reconcile_from_disk() ← restore extracted=True flags from note frontmatter │ ▼ scan_dropbox_pdfs() ← discover new PDFs; store RELATIVE-path keys │ ▼ get_unextracted_pdfs() ← filter: not extracted, not deferred, not terminal │ ▼ ProcessPoolExecutor ← persistent pool (max_workers=1); recycled on crash ├─ PyPDF2 extraction └─ OCR fallback ← tesseract v5.5 via pdftoppm → ~/.ocr_tmp/ │ ▼ write_note() ← atomic: tmp → os.replace() → os.fsync() │ ▼ save_extract_state() ← atomic; fires in finally: (poltergeist callback) │ ▼ _write_daily_report() ← self-contained markdown; survives agent exit

Key Design Decisions

Relative-path state keys — keys stored relative to DROPBOX_ROOT. Eliminates the cross-machine orphan class.

Poltergeist callback — finally: save_extract_state(state) fires even on SIGTERM. Verified by Layer Q.

Atomic writes — all state and note files use tmp → os.replace() → os.fsync(). No partial writes survive a crash.
